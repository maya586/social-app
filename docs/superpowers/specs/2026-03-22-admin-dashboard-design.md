# 服务端监控管理应用设计文档

## 概述

开发一个专业的、高端的、美观的管理后台应用，用于监控和管理社交应用服务端。

## 技术栈

- **客户端**: Flutter (Windows桌面应用)
- **服务端**: Go + Gin (扩展现有API)
- **UI风格**: Apple风格玻璃拟态 (与客户端统一)

## 功能需求

### 一、管理员登录

- 独立的管理员账号系统
- 用户名/密码登录
- JWT认证
- 记住登录状态

### 二、实时监控

监控以下指标：
- **服务器资源**: CPU、内存、磁盘使用率，网络流量
- **API性能**: 响应时间、每秒请求数、错误率
- **实时连接**: WebSocket连接数、在线用户数、通话房间数

### 三、用户管理

- 用户列表（分页、搜索、筛选）
- 用户详情查看
- 禁用/启用用户
- 查看用户聊天详情：
  - 用户基本信息
  - 发送/接收消息统计
  - 会话列表
  - 消息记录（按会话筛选）
  - 消息类型分布
  - 存储占用

### 四、首页驾驶舱

统计卡片：
- 总用户数
- 今日注册用户
- 在线用户数
- 总消息数
- 今日消息数

趋势图表：
- 7天用户增长趋势
- 7天消息发送趋势

快捷操作：
- 注册开关
- 异常用户提醒

### 五、系统设置

- 是否允许新用户注册（开关）
- 其他系统配置

## 数据库设计

### admins 表

```sql
CREATE TABLE admins (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username VARCHAR(50) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    nickname VARCHAR(50),
    role VARCHAR(20) DEFAULT 'admin',
    last_login_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### system_configs 表

```sql
CREATE TABLE system_configs (
    key VARCHAR(100) PRIMARY KEY,
    value TEXT NOT NULL,
    description VARCHAR(255),
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 初始配置
INSERT INTO system_configs (key, value, description) VALUES
('allow_registration', 'true', '是否允许新用户注册');
```

## API设计

### 管理员认证

```
POST   /api/v1/admin/login          # 管理员登录
POST   /api/v1/admin/logout         # 管理员登出
GET    /api/v1/admin/profile        # 管理员信息
```

### 仪表盘

```
GET    /api/v1/admin/dashboard      # 仪表盘统计数据
```

响应示例：
```json
{
  "total_users": 1000,
  "today_new_users": 15,
  "online_users": 42,
  "total_messages": 50000,
  "today_messages": 234,
  "active_conversations": 12,
  "user_trend": [10, 12, 8, 15, 20, 18, 15],
  "message_trend": [100, 120, 80, 150, 200, 180, 150]
}
```

### 系统监控

```
GET    /api/v1/admin/monitor        # 实时监控数据
GET    /api/v1/admin/monitor/stream # SSE实时推送
```

响应示例：
```json
{
  "server": {
    "cpu_percent": 45.2,
    "memory_percent": 62.5,
    "disk_percent": 35.8,
    "network_in_bytes": 1024000,
    "network_out_bytes": 512000

    "requests_per_second": 12.5,
    "avg_response_time_ms": 45,
    "error_rate": 0.02
  },
  "realtime": {
    "websocket_connections": 42,
    "online_users": 38,
    "active_call_rooms": 3
  }
}
```
